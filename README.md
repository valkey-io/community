# Valkey Community Resources
 
> Open source key/value datastore backed by the Linux Foundation. BSD licensed.
 
---
 
## Chat & Discussion
 
- **[Slack](https://valkey.io/slack)** — Primary real-time community hub. Use for questions, contributor coordination, and channel-specific topics (e.g. `#meetup-groups`).
- **[GitHub Discussions](https://github.com/orgs/valkey-io/discussions)** — Async forum for questions, proposals, and community conversation. Use when you want a searchable, threaded record.
---
 
## Code & Issues
 
- **[GitHub — Main Repo](https://github.com/valkey-io/valkey)** — Source code, PRs, releases. Use to browse code, open issues, or contribute.
- **[Bug Reports](https://github.com/valkey-io/valkey/issues/new?template=bug_report.md)** — Structured bug filing. Use when you've hit a reproducible issue.
- **[Feature Requests](https://github.com/valkey-io/valkey/issues/new?template=feature_request.md)** — Structured feature proposals. Use to pitch new capabilities to maintainers.
- **[Security Policy](https://github.com/valkey-io/valkey/blob/unstable/SECURITY.md)** — Responsible disclosure process. Use for potential CVEs or security concerns — do not file these as public issues.
---
 
## Social & Updates
 
- **[LinkedIn](https://www.linkedin.com/company/valkey/)** — Official company page. Use for project news and professional announcements.
- **[X / Twitter (@valkey_io)](https://x.com/valkey_io)** — Short-form updates and release announcements.
- **[Bluesky (@valkeyio.bsky.social)](https://bsky.app/profile/valkeyio.bsky.social/)** — Same as X but on Bluesky.
- **[Newsletter (Subscribe)](https://valkey.io/blog/valkey-newsletter-new/)** — Low-frequency email with releases, roadmap milestones, community highlights, and events. Use if you want a curated digest without monitoring social channels.

## Events & Meetups
 
- **[Events Calendar](https://valkey.io/events/)** — Official and third-party Valkey events. Use to find upcoming talks, conferences, and community gatherings.
- **[Meetup Groups](https://valkey.io/community/meetup-groups/)** — Charter and guidelines for local community meetups (free to attend, open to all). Use to find or start a local group; request a new one via a GitHub issue and post in `#meetup-groups` on Slack.
---
 
## Content & Learning
 
- **[Blog](https://valkey.io/blog/)** — Official posts: release announcements, technical deep dives, how-tos, and community highlights. Use to understand new features or get production guidance.
- **[Documentation by Topic](https://valkey.io/topics/)** — Organized concept docs (data types, clustering, replication, security, etc.). Use when learning or referencing a specific area.
- **[Command Reference](https://valkey.io/commands/)** — Full command API including first-party modules. Use as the authoritative reference for command syntax and behavior.
- **[Client Libraries](https://valkey.io/clients/)** — Official clients for Python, Java, Go, Node.js, PHP, C#. Use to find the right SDK for your language.
- **[Performance Dashboards](https://valkey.io/performance/)** — Throughput benchmarks across versions. Use to validate regressions or compare release performance.
- **[Try Valkey (online)](https://valkey.io/try-valkey/)** — Browser-based sandbox. Use for quick experimentation without a local install.

## Governance & Conduct
 
- **[Code of Conduct](https://valkey.io/code_of_conduct)** — Community behavior standards (LF Events CoC). Use if you experience or witness misconduct.
- **[Leadership](https://valkey.io/leadership)** — Current maintainers and project leads.
- **[About / FAQ](https://valkey.io/topics/faq)** — Project background, licensing, and common questions.
- **[Trademark Policy](https://lfprojects.org/policies/trademark-policy/)** — Rules for using the Valkey name and logo.
- **[Brand Guidelines](https://github.com/valkey-io/assets/blob/main/Valkey%20Branding/valkey_brand_guidelines.pdf)** — Logos, colors, and usage rules. Use when creating meetup slides or community content.

2. Browse [open issues](https://github.com/valkey-io/valkey/issues) on GitHub
3. Read the [Security Policy](https://github.com/valkey-io/valkey/blob/unstable/SECURITY.md) before reporting vulnerabilities
4. Follow the [Code of Conduct](https://valkey.io/code_of_conduct)
